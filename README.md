## 📝 Description

**GitHub Analytics Dashboard** is a real-time repository health monitoring tool built for maintainers and engineering teams. It fetches live data directly from the **GitHub REST API**, processes key repository metrics, and presents them through a clean, interactive **Streamlit** interface — complete with AI-powered recommendations to help teams act on what matters most.

> 🤖 This project was rapidly prototyped using **[Zerve's Agentic AI](https://zerve.ai)** — an AI-native data & analytics development environment that accelerates end-to-end data workflows from raw data to deployed applications.

---

## ✨ Features

- **📈 KPI Cards** — At-a-glance metrics including open issues, stale issues, merged PRs, average issue close time, and average PR merge time
- **📊 Four Interactive Charts** — Visualize issue age distribution, PR merge time distribution, top issue labels, and contributor activity across the repository
- **⏳ PR Wait Time Table** — Ranked table of open pull requests that have been waiting the longest for review, helping prioritize code review efforts
- **🤖 AI Recommendations** — Intelligent, context-aware suggestions generated by an LLM based on live repository health data to guide maintainer actions
- **⚡ Response Caching** — GitHub API responses are cached to reduce redundant requests and improve dashboard load performance
- **🌐 Public Repo Support** — Works out-of-the-box with any public GitHub repository — no authentication required for public data

## 🚀 How to Run

### Prerequisites
- Python 3.9+

### Local Setup
1. Clone the repository
git clone https://github.com/your-org/github-analytics-dashboard.git
cd github-analytics-dashboard
2. Install dependencies
pip install -r requirements.txt
3. (Optional) Set your GitHub token as an environment variable
export GITHUB_TOKEN=your_personal_access_token_here
4. Run the Streamlit app
streamlit run app/main.py

The dashboard will open automatically in your browser at `http://localhost:8501`.

> 💡 **Tip:** To analyze a different repository, update the `REPO` variable in the sidebar or configuration file (e.g., `owner/repo-name`).
